# 稳定测试里程碑 2026-04-10

## 定位
这是 `ParallelTurnPvp` 当前第一个可回滚的双机稳定测试点。

## 已验证主链
1. 多人菜单进入 `ParallelTurn PvP`
2. 主机/客机通过 `DirectConnectIP` 进入同一调试对局
3. 发放固定牌组、药水、遗物
4. 双方正常出牌、喝药、选目标
5. 正常结束回合并进入下一回合
6. 正常触发胜负结束
7. 当前版本未再复现房间因 `StateDivergence` 直接断开

## 当前稳定边界
1. PvP 规则仍然建立在原版即时战斗壳之上
2. `BattlewornDummy` 仍是战斗外壳，不是最终规则宿主
3. 有限意图已经有 runtime 和首版 overlay，但还不是最终美术/交互形态
4. 最早结束回合奖励的 `+3 治疗` 设计保留，但 live combat 实现已暂时禁用

## 关键结论
1. 不要再直接把 PvP authoritative result 写回原版 live combat state
2. 任何额外 PvP 规则奖励都必须走“原版可同步动作”或未来的“延迟结算层”
3. 当前版本适合作为下一阶段开发基线

## 下一阶段建议
1. 把有限意图从首版 overlay 推进成正式 UI
2. 把首锁奖励改造成真正的同步动作，再恢复规则
3. 逐步把规则执行权从原版即时战斗转移到 `PvpActionLog + PvpRoundResolver`

## 同步修复补记
1. 已验证 ConsoleCmdGameAction("heal ...") 不是安全的联机同步实现。
2. 这条路径会在客户端触发 RecordInitialState must be called first，随后破坏 replay/checksum 链并导致 StateDivergence。
3. 因此当前稳定基线仍保持：最早结束回合奖励的 +3 治疗 设计保留，但 live combat 实现继续禁用。
4. 下一次恢复这条规则时，必须换成真正安全的同步动作实现，或直接等延迟结算层接管。
