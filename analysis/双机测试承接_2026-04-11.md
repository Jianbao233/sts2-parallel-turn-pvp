## 本地动态测试结果
- host 端关键日志：
  - `[ParallelTurnPvp] Requested debug host flow from multiplayer submenu.`
  - `[DirectHost] DirectHost started on port 33771 with ID 76561198679823594`
- join 端关键日志：
  - `[ENetClient] Sending handshake with net ID 1001`
  - `[ENetClient] Received net ID (76561198679823594) during handshake that did not match ours!`
  - `[JoinFlow] Connection failed: DisconnectionReason InternalError False`
- 结论：
  - 本地 `--fastmp=join -clientId=1001` 使用的是假 net ID
  - 当前 host 通过 DirectConnectIP DirectHost 暴露的是主机真实 SteamID 身份
  - 两者不是同一套身份路径，因此 join 失败

## 为什么下一轮应切回真实双机
- 已有旧客机日志表明，真实双机直连走的是：
  - `DirectClientConnectionInitializer netId=<SteamId> host=<ip>:33771`
- 这条链路和本地 fastmp `ENetClientConnectionInitializer(netId=1001, ip=127.0.0.1, port=33771)` 不是一回事。
- 因此，当前项目要验证“能否真实加入主机房间并进入 lobby”，应回到双机 `DirectConnectIP` 路线，而不是继续对本地 fake client-id 做过度优化。
