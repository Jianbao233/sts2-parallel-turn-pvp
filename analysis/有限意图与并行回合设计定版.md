# 有限意图与并行回合设计定版

## 1. 目标
1. 方案固定为 `面对面同房间 + 并行规划 + 回合末统一结算`。
2. 房间继续作为表现层保留，后续规则执行权逐步迁移到 `PvpActionLog + PvpRoundResolver + Host authoritative result`。
3. 当前 `v0` 仍允许即时执行外壳存在，但后续改造方向已经锁定，不再走“实时公开完整牌名和目标”的路线。

## 2. 有限意图规则
### 2.1 动作类型公开层级
- 定版为 `B` 档：公开动作大类，不公开具体牌名。
- 首版动作大类固定为：
  - `Attack`
  - `Guard`
  - `Buff`
  - `Debuff`
  - `Summon`
  - `Recover`
  - `Resource`

### 2.2 对方信息揭示规则
- 每当你打出一张牌，就解锁一次对方本回合已提交动作的意图信息。
- 解锁预算只由 `PlayCard` 累计，不由药水或结束回合累积。
- 可见信息按对方提交顺序展开。
- 若你本回合打出 `N` 张牌，则你最多看到对方前 `N` 个动作槽位的大类意图。

### 2.3 公开的能量信息
- 只公开 `本回合开始时` 对方拥有的能量。
- 不公开对方当前剩余能量。
- 不公开对方回合中途能量变化。

### 2.4 结束回合规则
- `结束回合` 一旦按下即不可撤销。
- 状态机需要显式表现该玩家已结束回合。
- 本回合最早锁定的玩家获得 `3` 点生命恢复。
- 该数值当前为临时定值，后续可再平衡。

### 2.5 目标公开规则
- 不公开精确目标。
- 只公开目标侧：
  - `Self`
  - `Enemy`
- 不公开是 `Hero` 还是 `Frontline`。

## 3. 玩家侧表现流
1. 回合开始时，双方同时进入规划阶段。
2. 玩家打牌/喝药时，不向对面公开牌名，只在意图轨道中累积一个动作槽位。
3. 你每打出一张牌，就获得一次“查看对方下一格公开意图”的预算。
4. 双方都按下结束回合后，Host 使用同一份起始快照和双方日志统一结算。
5. 结算结果以单独的表现时间轴播放，不再依赖“我出一张牌，对面立即掉血”的即时战斗语义。

## 4. 数据结构定版
- `PvpIntentCategory`
- `PvpIntentTargetSide`
- `PvpPublicIntentSlot`
- `PvpPlayerIntentState`

### 4.1 `PvpPlayerIntentState` 必须包含
1. `PlayerId`
2. `RoundStartEnergy`
3. `Locked`
4. `IsFirstFinisher`
5. `Slots`

### 4.2 `PvpPublicIntentSlot` 必须包含
1. `Category`
2. `TargetSide`

## 5. 实现顺序
1. 先在 runtime 中接入有限意图数据结构、锁定状态和首锁回血。
2. 再把意图通过同步层和 UI 展示出来。
3. 最后把实际卡牌/药水执行从“即时执行后记日志”改成“先记日志，回合末统一结算”。

## 6. 当前决定
1. 不做“实时公开完整牌名”。
2. 不做“实时公开精确目标”。
3. 面对面房间方案继续保留。
4. 后续所有并行规划工作以本定版为准。
